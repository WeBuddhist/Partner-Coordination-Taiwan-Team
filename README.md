## Document Hierarchy

This plan is organized in four levels:

1. **Milestones** — quarter-level and sprint-level date markers (map directly to GitHub Milestones).
2. **Epics** — large bodies of work scoped inside a milestone (`epic` label in GitHub).
3. **User Stories / Tasks** — individual issues. User stories use the *As a … I want … so that …* format. Pure operational items are **Tasks**.
4. **Subsections** — Goal · Acceptance Criteria · Sub-tasks · Notes · DAB tie-in.

### Suggested Labels
`epic` · `user-story` · `task` · `sprint-1` · `pillar-1` · `pillar-2` · `partner` · `outreach` · `research` · `drive` · `content` · `design` · `video` · `priority:high` · `priority:medium` · `blocked` · `needs-review`.
